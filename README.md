# HateXScore: A Metric Suite for Evaluating Reasoning Quality in Hate Speech Explanations

A public repository containing code, evaluation scripts, and resources for the paper **"HateXScore: A Metric Suite for Evaluating Reasoning Quality in Hate Speech Explanations"**. HateXScore is a four-component metric suite for evaluating the **reasoning quality** of explanations generated by hate speech detection systems, going beyond standard label-based metrics such as Accuracy and F1.

## Overview

Hate speech detection systems are often evaluated only by whether they predict the correct label. However, such metrics do not reveal whether the model’s explanation is faithful, complete, and logically coherent. HateXScore addresses this gap by evaluating model explanations across four complementary dimensions: conclusion explicitness, quotation faithfulness, protected-group identification, and internal consistency. It is designed as a **diagnostic complement** to traditional classification metrics rather than a replacement for them. 

HateXScore is especially useful in cases where dataset labels are ambiguous or debatable. A model may disagree with the gold label while still providing a strong and well-grounded explanation. In such cases, HateXScore can help surface potentially unreliable annotations and reasoning-quality differences that standard evaluation would miss.

## ✨ Key Features

- A new metric suite for evaluating explanation quality in hate speech detection.  
- Goes beyond **Accuracy** and **F1** by explicitly scoring explanation reasoning.  
- Four sub-metrics:
  1. **HTC (Hate-Type Check)**: whether the explanation clearly states hateful / non-hateful.
  2. **QF (Quotation Faithfulness)**: whether quoted evidence is meaningful and causally affects prediction.
  3. **TGI (Target-Group Identification)**: whether the explanation correctly identifies the attacked protected group.
  4. **CC (Consistency Check)**: whether prediction, evidence, and target-group identification are logically consistent.  
- Supports **policy-configurable** protected-group inventories.  
- Supports multilingual evaluation with language-specific tokenization for **English / Chinese / Korean**.  
- Helps identify **interpretability failures** and **annotation inconsistencies** in hate speech datasets.
